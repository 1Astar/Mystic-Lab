## 1. 结论

- **提问页默认仍用本地规则**即时给角度；另加**可选**「让 AI 帮我优化」小按钮（hover 文案），不自动调 API。
- **结果页**底部加「对结果有疑问？可能是问法不对」入口，可 AI 改问，并支持「仅保存参考」与「采用并重新占问」。
- **不满意反馈**双写：本机注入下次 prompt；同步 Star PM Ideas 收件箱，归属随心而行。

实现顺序：**先扩本地规则，再共改写模块 + 两处 UI + 反馈双写。**

---

## 2. 背景与问题

当前「优化问题」全部来自 `src/tarot/question-coach.ts` 规则模板，与「AI 解读」配置无关。用户配置 API 后期望提问被优化，造成认知落差；且部分求职问法（如「找到下一份工作」）未命中 `找工作` 正则，落回三条泛模板。

## 3. 目标与非目标

### 目标

1. 覆盖常见封闭式场景的本地角度（求职、情感、职场、学业、自我/金钱等）。
2. 提问页、结果页均可按需调用 AI 生成更开放式问法。
3. 负反馈可立即影响本机下次改问，并落入 Star PM 供产品迭代。

### 非目标（首版不做）

- 输入时实时 / 自动调 AI
- 模型微调或服务端 fine-tune
- 不经用户确认的自动重抽
- Mystic Lab 自建独立反馈后端

## 4. 架构

```
┌───────────── 提问页 ─────────────┐     ┌──────────── 结果页 ────────────┐
│ 规则教练 analyzeQuestion（现有） │     │ 解读 tabs（现有 AI 解读）       │
│ [AI] 小按钮 → QuestionRewritePanel│     │ 问法疑虑入口 → 同一 Panel       │
└──────────────┬───────────────────┘     └──────────────┬─────────────────┘
               │                                        │
               ▼                                        ▼
        rewriteQuestionViaLlm (共用) ← loadAiSettings + 本机负反馈 prompt 片段
               │
               ├─ 成功：展示 3 条候选 + 重新生成
               └─ 不满意 → recordFeedback
                        ├─ localStorage（即时）
                        └─ POST Star PM /api/ideas/capture（产品侧）
```

### 模块边界

| 单元 | 职责 | 依赖 |
|---|---|---|
| `question-coach.ts` | 规则识别 + 角度模板 | 无 AI |
| `ai/question-rewrite.ts`（新） | 调 LLM 生成 3 条开放式问题；JSON 解析与校验 | `llm-client` / settings |
| `ai/question-feedback.ts`（新） | 本机读写；拼 prompt 记忆；上报 Star PM | settings 中的 Star PM 配置 |
| `ui/question-rewrite-panel.ts`（新） | 共用面板：生成 / 再生 / 选用 / 反馈 | 上述两模块 |
| 提问页 / 结果页 | 挂载入口与「采用」后行为差异 | panel |

## 5. 本地规则扩展（第一刀）

文件：`src/tarot/question-coach.ts`（及必要时 `question-guide.ts` 封闭判定）。

### 5.1 匹配修复

`generic_closed` 下求职分支关键词由仅 `找工作` 扩为至少：

`找工作|找到.*工作|下一份工作|新工作|失业|换工作|跳槽|求职|offer|录用`

使「什么时候能找到下一份工作」进入求职专用角度，而非三条泛模板。

### 5.2 新增 / 细化 pattern（首版清单）

在现有 `love_*` / `interview` / `offer` / `quit` 之上补充（命名可按实现微调，须有测例）：

| pattern 意向 | 触发信号（示例） | 角度方向 |
|---|---|---|
| job_search | 找工作、下一份工作、求职、什么时候能找到 | 阻碍 / 两面 / 行动 |
| promotion | 升职、加薪、绩效 | 看清什么 / 卡点 / 行动 |
| team_conflict | 同事、领导、汇报、冲突 | 关系动态 / 我能调整什么 |
| love_contact | 会不会联系、要不要找对方 | 联结 / 边界 / 下一步 |
| love_third | 第三者、劈腿嫌疑（封闭） | 看清事实倾向 / 我该如何面对（避免罪名式预言） |
| study_exam | 考试、过不过、分数 | 准备重点 / 心态 / 意义 |
| study_path | 志愿、读不读研、换专业 | 真实需要 / 风险 / 两面 |
| money | 理财、投资、买不买 | 风险与需要 / 看清什么（不做荐股） |
| move_city | 搬家、换城市、出去 | 动机 / 风险 / 适应 |
| anxiety_decide | 纠结、要不要、该不该（泛） | 本质 / 调整 / 两面 |

未命中仍走 `generic_closed` 或 `open`。

### 5.3 验收

输入「什么时候能找到下一份工作」→ 出现求职向角度（含直接问），**不是**仅「我现在需要看清什么？」等三条泛化句。

## 6. AI 改问（提问页 + 结果页）

### 6.1 前提

- `isAiConfigured()` 为真方可请求；否则短提示并引导打开「AI 解读」配置。
- 使用用户已配置的 OpenAI 兼容接口与模型。

### 6.2 输出约定

LLM 返回严格 JSON（无 markdown 围栏优先；若有围栏则剥离）：

```json
{
  "candidates": [
    { "label": "看阻碍", "question": "……？" },
    { "label": "看行动", "question": "……？" },
    { "label": "看两面", "question": "……？" }
  ]
}
```

硬性要求（写入 system/user prompt）：

- 恰好 3 条；必须比原问更开放，避免「会不会 / 什么时候一定 / 吗」式封闭收束
- 保留原问主题与人称，不跑题
- 中文；每条问话 ≤ 40 字为宜
- 结合本机近期负反馈摘要（若有），避免重复同类错误

解析失败：提示重试；不写入输入框。

### 6.3 提问页 UI

位置：`question-head` 内，靠近「怎么问更好？」。

- 小按钮（图标或「AI」），`title` / 自定义 tooltip：**让 AI 帮我优化**
- 点击后在输入框与规则教练之间展开 `QuestionRewritePanel`
- 点选候选 → 写入 `#tarot-question` 并 `dispatchEvent('input')`，规则教练同步刷新
- 提供：重新生成、不满意并反馈、收起
- **不**提供「重新占问」（尚在提问阶段）

### 6.4 结果页 UI

位置：结果区底部、反思框附近（`result-panel` 行动区之上或并列）。

文案入口：**对结果有疑问？可能是问法不对 — 让 AI 帮你改问**

面板同构，另增选用后行动：

| 操作 | 行为 |
|---|---|
| 仅保存参考 | 写入本机改写历史；本次解读不变 |
| 采用并重新占问 | 将 `question` 设为所选文案，走现有重来流程回到牌阵/抽牌（与「再问一次」同类，但预填新问题） |

### 6.5 重新生成

同一面板内「重新生成」再次调用；请求中带：

- 原问题
- 上一轮候选（避免原样重复）
- 本机负反馈最近 N 条（默认 5）

## 7. 反馈双写

### 7.1 触发

面板内 **不满意并反馈原因**：

- 多选标签：`太空泛` / `仍太封闭` / `偏题` / `太长` / `不够具体` / `其他`
- 选填短文（可选，≤ 200 字）
- 附带上下文：原问题、当轮候选、来源页（`question` | `result`）、时间戳

### 7.2 本机

- `localStorage` key：`mystic-lab-question-rewrite-feedback`
- 结构：数组，新在前，上限例如 30 条；prompt 只用最近 5 条
- 立即生效，无需网络

### 7.3 Star PM

- `POST {starPmBaseUrl}/api/ideas/capture`
- Header：`x-ideas-capture-secret` 或 `Authorization: Bearer …`（与 Star PM `IDEAS_CAPTURE_SECRET` / `CRON_SECRET` 一致）
- Body 字段约定：

| 字段 | 值 |
|---|---|
| `title` | `[问法反馈] ` + 原问题截断（≤ 40 字） |
| `type` | `内容想法`（或实现时统一枚举名） |
| `relatedProjectId` | `proj-moonpie` |
| `summary` | 标签拼接 + 一句话 |
| `rawThought` | 结构化全文（原问题、候选 JSON、标签、短文、来源页、客户端时间） |
| `priority` | `P2` |
| `suggestedNextStep` | `对照反馈改 question-coach 规则或改问 prompt` |

### 7.4 Star PM 客户端配置

在 AI 设置弹窗增加「产品反馈同步（可选）」：

- Star PM Base URL
- Capture Secret（仅本机保存，与 LLM Key 分字段）

未配置 Secret 时：本机仍保存；UI 提示「已保存在本机；配置 Star PM 后可同步到收件箱」。已配置但网络失败：本机成功优先，展示非阻断错误。

环境变量（构建时可选，与 UI 互为补充，以实现计划为准）：如 `VITE_STAR_PM_BASE_URL`；Secret **禁止**打进前端默认值，须用户本机填或本地 `.env` 仅开发用且不入库。

## 8. 错误与边界

| 情况 | 行为 |
|---|---|
| 输入为空或过短 | AI 按钮禁用或点击提示先填写问题 |
| AI 未配置 | 引导配置，不发起请求 |
| LLM 4xx/5xx / 超时 | 面板内错误文案 + 可重试 |
| JSON 非法 | 同失败，可重试 |
| Star PM 401 | 本机已存；提示检查 Secret |
| CORS | 浏览器直调 Star PM 若跨域失败：在 Star PM capture 路由补 CORS（允许 Mystic Lab 部署域名），或首版仅本机保存并在设置页提供「复制反馈 JSON」兜底。实现计划须二选一写死，默认优先补 CORS。 |

- 规则：求职句「什么时候能找到下一份工作」命中 job 角度；至少 1～2 个新 pattern 的单测
- 改写：mock LLM 返回合法 / 非法 JSON 的解析行为
- 反馈：本机写入可读取；Star PM 请求在配置缺失时不抛未捕获异常
- UI 冒烟：提问页填回输入框；结果页两步采用均可点（e2e 可选）
