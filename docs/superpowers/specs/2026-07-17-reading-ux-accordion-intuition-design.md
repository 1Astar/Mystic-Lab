## 1. 结论

- **解读 Tab 用折叠（Accordion）降信息过载**：默认只展开「热点整体解读」与「给你的一个提问」；「现实状况 / 行动指南 / 继续追问」默认收起。
- **答案倾向去量化标签**：不再展示「偏低 / 中性偏负」等 overall/tendency 文案；只保留一句话答案 + 关键提醒，并支持追问「该付出什么行动来弥合」。
- **补牌有明确主题引导**，不再只有「还想补一张？」。
- **每张牌：翻开 →（可选背景）→ 写第一直觉 → 再揭开 AI/内置解读**；解读页对照展示直觉与 AI；可跳过。AI 可在写直觉时后台预跑（方案 1）。
- **深色卡片对比度提高**，保证手机上可读。

---

## 2. 背景与问题

当前「结合你的问题」把热点解读、元素映射、行动、追问堆在同一滚动面，手机滑动成本高。  
「你的答案倾向」里的「偏低 / 中性偏负」来自规则 `buildAnswerTendency`（正逆位 + 关键词），**不是 AI 算分**，却易被读成判决。  
补牌入口文案偏弱。  
Slogan「答案不在牌里，在你心里」缺少「先直觉、再对照」的仪式落点——现流是翻牌后直接出完整解读。

---

## 3. 目标与非目标

### 目标

1. 首屏解读获得感更高、滑动更短。
2. 倾向模块不再用分数感标签制造焦虑。
3. 补牌意图清晰（主题 + 建议牌/行动牌）。
4. 每张牌可先写直觉再看解读，并在解读页对照。
5. 深色卡片文字可读性提升。

### 非目标

- 不做真实数值评分模型。
- 不改牌阵/抽牌仪式手势主流程。
- 不强制每张牌必须写直觉（可跳过）。
- 本轮不重做结果页总览信息架构（仅卡片级解读 Tab + 补牌文案）。

---

## 4. 用户流程（B）

```
翻牌完成
  → [仅首张/本轮一次] preReading 可选背景（保持现状）
  → cardIntuition（新）：展示牌面身份（名/正逆）+ 输入框
       「翻开后、看解读前：你的第一直觉是什么？」
       [跳过] [写下后看解读]
  → （可选）后台已跑完 interpret；否则此处等待
  → cardReview：现有 Tab；解读 Tab 含直觉对照 + Accordion
```

多张牌：每张抽→翻→**直觉**→解读→收下→下一张（与现 `currentIndex` 节奏一致）。  
补牌：补抽后同样走直觉再解读。

### AI 时机（已拍板：方案 1）

- 进入 `cardIntuition` 时即可 `void` 启动 `finishFlipInterpret` 的 interpret 部分（不立刻进 cardReview）。
- 用户提交/跳过直觉后：若 interpret 已完成 → 直接 `cardReview`；未完成 → 短 loading，再 `cardReview`。
- 直觉文本写入 `CardReading.userIntuition`（或等价字段），再进解读页。

---

## 5. 数据模型

在 `CardReading`（`src/interpretation/types.ts`）增加：

```ts
/** 翻牌后、看解读前用户写下的第一直觉；可空（跳过） */
userIntuition?: string;
```

- Journal / progress snapshot：随现有 `reading.cards[]` 一并持久化（不另开表）。
- Replay / 图鉴遭遇：本轮**不强制**展示直觉；结果页解读 Tab 有即可。若 replay 已复用同一 `CardReading`，自动带上。

`AnswerTendency` 类型可保留 `overall` / `tendency`（规则仍可生成，便于日后调试），**UI 不再渲染这两项**。

## 6. UI 规格

### 6.1 Accordion（解读 Tab ·「结合你的问题」）

| 区块 | 默认 |
|---|---|
| ① 热点整体解读 | **展开** |
| ② 现实状况（元素映射 / 其余情境段） | 折叠 |
| ③ 行动指南（建议/行动/疏导段） | 折叠 |
| 继续追问（chips + 已答追问） | 折叠 |
| 「给你的一个提问」整卡 | **保持展开**（独立 section，不进折叠组） |

交互：

- 折叠头：标题 +「展开」/「收起」；整行可点。
- 仅用 CSS/`<details>` 或轻量 JS toggle；不引入组件库。
- 「这张牌在说什么」标准牌义、答案倾向（去标签后）仍在折叠组之外、默认可见。

### 6.2 答案倾向

展示：

- 标题：你的答案倾向  
- 一句话答案  
- 关键提醒  
- **不展示**「整体判断」「结果倾向」

追问：在「继续追问」chips 前或同列，固定增加一条（有 `actionTip` 时）：

- 「我该付出什么样的行动来弥合？」  
- 走现有 `interpretCardFollowUp` 链路。

### 6.3 第一直觉中间页

- Lead：牌已翻开 · 先听自己，再听解读  
- 展示：牌名 + 正/逆（可小图，若现有组件易用则复用；否则文字即可）  
- Textarea：placeholder 例「看到这张牌，你心里第一个念头是…」  
- 主按钮：看解读（有内容则保存；空则等同跳过或提示可跳过——**空内容允许直接进解读**）  
- 次按钮：跳过，直接看解读  

解读 Tab 对照块（在「结合你的问题」之上或答案倾向下）：

```
你的第一直觉
「……」
（跳过则整块不渲染）
```

不自动把直觉拼进 LLM prompt（本轮）；避免改变既有解读缓存语义。若后续要「对照强化」，另开需求。

### 6.4 补牌引导

当可补牌时：

- 按钮文案可仍为「还想补一张？」或改为「补一张建议/行动牌」  
- **必有**辅助说明（按钮下或 `teach-hint`）：  
  `如果这轮还不够清晰，可针对【{主题}】再补一张建议牌或行动牌`  
- `{主题}` 来自 `detectQuestionTheme(question)` 的展示名（工作→面试求职/工作；感情；学业；自我）。面试类问题可用更贴的「面试求职」：若问题命中面试关键词则用「面试求职」，否则用主题展示名。

### 6.5 对比度

`.reading-layer-card`：

- 背景由约 `rgba(46, 27, 75, 0.45)` 调整为约 `rgba(46, 27, 75, 0.78)`（或等价提高不透明度）  
- `border` 略提亮（如 `var(--line)` → 略高 alpha）  
- 答案倾向卡保持金边区分，背景同步略提亮  

目标：白字/浅紫字在深紫底上更易读；不改为浅色主题。

## 8. 验收标准

1. 手机宽：解读 Tab 默认首屏可见热点整体 + 提问；现实状况/行动/追问需点开。  
2. 答案倾向无「偏低」「中性偏负」等标签字段。  
3. 翻牌后先到直觉页，跳过或提交后才见完整「结合你的问题」AI/内置文。  
4. 写下的直觉出现在解读对照块；journal 重进仍在。  
5. 可补牌时可见带【主题】的引导句。  
6. 解读卡片背景明显比改前更实，暗屏下正文可读。

---

## 9. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 多一步导致流失 | 跳过一键；文案强调「先听自己」 |
| 后台 AI 失败 | 与现网一致回退内置解读；直觉仍保留 |
| Accordion 可访问性 | 使用 button + `aria-expanded`，或原生 `<details>` |

---

## 10. 实现顺序建议

1. 对比度 + 倾向去标签 + 补牌文案（低风险）  
2. Accordion  
3. `userIntuition` + `cardIntuition` 状态与对照块  
4. 弥合追问 chip  
5. 手工走查多牌 + 补牌 + 跳过直觉
