## 1. 结论

- **交互**：深度签内展示完整装卦表；点某一行/爻展开**实盘卡**。现状签 / 核心签不改。
- **时间**：默认当前本地时间；结果页顶栏可改「占卜时间」；一改则重算四柱与六神（装卦表与打开中的卡同步刷新）。
- **四柱**：用已有依赖 `lunar-javascript`（`Solar` → `Lunar`）取年月日时干支与日干；空亡若库可取则展示，否则本批不做神煞堆砌。
- **装卦**：自写京房纳甲（爻地支 + 五行）+ 以**卦宫五行**定六亲 + 按**日干**自初爻上排六神。
- **不做**：元神/忌神星图（P3）、起卦页强制选时（可下一批）、神煞大全、把装卦塞进四层主解读正文。

## 4. 数据模型

### 4.1 占卜时刻

```ts
/** ISO 或 Date；结果页 state，不强制写入 CastResult */
castAt: Date
```

手札保存：本批可将 `castAt` ISO 字符串写入 journal entry（若改动过小可只存结果页即时算，不入库——实现时优先写入，便于复盘）。

### 4.2 四柱视图

```ts
type SiZhu = {
  year: string;  // 如 丙午
  month: string;
  day: string;
  hour: string;
  dayStem: string; // 日干，如 壬
  xunKong?: string; // 空亡，可选
}
```

来源：`Solar.fromDate(castAt).getLunar()` 的 `getYearInGanZhi` / `getMonthInGanZhi` / `getDayInGanZhi` / `getTimeInGanZhi`；日干取日柱首字。空亡：扩展类型声明后调用库方法（若无则省略字段）。

**子时**：跟随 `lunar-javascript` 对当日时刻的划分，不自造边界规则。

### 4.3 装卦行（本卦六爻，自下而上 index 0–5）

```ts
type YaoDress = {
  index: number;
  label: string;       // 初爻…
  bit: 0 | 1;
  changing: boolean;
  branch: string;      // 地支
  wuxing: WuXing;      // 支五行
  liuqin: '兄弟' | '子孙' | '妻财' | '官鬼' | '父母';
  liushen: string;     // 青龙…白虎…
  isShi: boolean;
  isYing: boolean;
  /** 动爻时：变卦同位地支等，可选 */
  changedBranch?: string;
  changedWuxing?: WuXing;
}
```

聚合：`dressHexagram(cast, siZhu) → { palace, palaceWx, rows: YaoDress[] }`。

### 4.4 纳甲规则（实现契约）

1. **八卦纳支**（标准京房表，自下而上三爻）：  
   - 乾：子寅辰；坤：未巳卯；震：子寅辰；巽：丑亥酉；  
   - 坎：寅辰午；离：丑亥酉；艮：辰午申；兑：巳卯丑。  
2. **本卦六爻地支** = 下卦三支 + 上卦三支。  
3. **卦宫**：由现有八宫表（`hexagrams.ts` 的 `PALACES`）反查本卦所在宫 → 宫五行（乾兑金、震巽木、坎水、离火、坤艮土）。  
4. **六亲**：以宫五行 vs 爻支五行——  
   - 同我→兄弟；我生→子孙；我克→妻财；克我→官鬼；生我→父母。  
5. **六神**：日干 → 初爻起神，依次上排。映射：  
   - 甲乙→青龙；丙丁→朱雀；戊→勾陈；己→腾蛇；庚辛→白虎；壬癸→玄武。  
   （与常见排盘软件一致；实现时用单测钉死日干样例。）

变卦：对 `changedLines` 再纳一次支（仍用变卦上下卦），动爻行写入 `changedBranch`。

## 5. UI

### 5.1 深度签结构（替换占位）

1. **占卜时间**：`<input type="datetime-local">` + 文案「默认此刻，可改」  
2. **四柱条**：`年柱 · 月柱 · 日柱 · 时柱`；旁注日干；空亡若有则显示  
3. **装卦表**：六行（建议自上爻到初爻展示，与竞品一致，或自下而上与教学一致——**实现取自上而下展示，与常见排盘一致**），列：六神｜六亲｜支+五行｜世/应｜动标记  
4. **点行**：展开/浮层实盘卡；再次点同行关闭；换行切换  

### 5.2 实盘卡内容

- 标题：爻位 + 阴阳 + 是否动/世/应  
- 干支五行、六亲（白话一句）、六神（象义一句）  
- 与问题映射：复用 `resolveYongShen`——若该爻六亲=用神则高亮「本题相关」  
- 动爻：注明变后地支  
- 📖：沿用现有古文 hint，不在此塞全文  

### 5.3 绑定

`bindResultLayers` 扩展：tab 切换、datetime change → 重渲深度面板、行点击开卡。  
重渲时保持当前 tab=`deep`。

## 7. 验收标准

1. 进入结果页深度签：见默认四柱与六行装卦，非占位文案。  
2. 改占卜时间（尤其跨日干）：六神列变化；四柱更新。  
3. 点一行出实盘卡；卡内六亲/干支与表一致；用神相关爻有提示。  
4. 已知样例卦（如震宫豫）纳支与六亲符合京房常识（单测）。  
5. `tsc` 与相关 vitest 通过。
