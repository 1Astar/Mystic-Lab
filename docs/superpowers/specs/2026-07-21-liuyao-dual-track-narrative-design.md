# 六爻结果页：双轨文案引擎 — 设计规格

| 项 | 内容 |
|---|---|
| 项目 | Mystic Lab · 随心而行 |
| 日期 | 2026-07-21 |
| 状态 | 待实现 |
| 范围 | 六爻结果页（速断 / 学习）文案与信息架构 |

## 结论

用**一层共享事实 + 两套渲染器**重写六爻结果叙事：

- **速断**：表格式盘面 + 短结论三 Tab（结论｜分类建议｜原文）——只「看结果」
- **学习**：完整「因为…所以…」因果链 + 生活映射 + 实战指令（含变卦选择题）

禁止只点名概念（「这里有世应」）；可以写「世=你」作锚，但必须立刻映射到**这一卦、这一爻位**对用户生活的含义。

## 目标与非目标

### 目标

1. 任意起卦均可数据驱动生成因果链（非单卦样板写死）
2. 速断与学习共用同一套事实，避免两套互相矛盾的说法
3. 学习页把「教科书步骤」改成「指令 + 本卦实战」；变卦实战用选择题再揭释义
4. 深度页补齐教学骨架：生克星图原理解释、古文三层、FAQ 推理路径

### 非目标

- 不引入 LLM 现场生成主文案（保持可测、可版本化）
- 不重做起卦仪式 / 纳甲算法本身
- 不一次做齐六爻全连、月破库墓等进阶断法
- 不改塔罗 / 小六壬结果页

## 模式分流

| | 速断 | 学习 |
|---|---|---|
| 用户意图 | 看结果 | 学会读 |
| 盘面 | 仿传统：本卦 ∥ 变卦表（六亲 / 世应 / 动爻；可含地支五行若已装卦） | 可沿用 SVG 主视觉；深度 Tab 继续装卦表 |
| Tab | `结论`｜`分类建议`｜`原文` | `此刻解读`｜`核心要素`｜`怎么读卦`｜`深度推演` |
| 文案密度 | 短句、无长推导 | 锚点 → 推导 → 结论 → 行动；映射到生活 |

## 共享事实层 `ReadingFacts`

一次起卦只算一次，供双轨渲染：

```ts
type ReadingFacts = {
  question: string;
  domain: 'career' | 'love' | 'life' | 'general';
  primary: { name: string; fullName: string; keywords: string[]; gist: string };
  changed: { name: string; fullName: string; keywords: string[]; gist: string } | null;
  upperNature: string; // e.g. 山
  lowerNature: string;
  shi: { line: number; label: string; role: string }; // role = 爻位角色：基础/关口/…
  ying: { line: number; label: string; role: string };
  shiYingRel: { rel: string; verdict: string; tip: string };
  changing: { indexes: number[]; labels: string[] };
  yong: { name: string; why: string; tip: string; matchedLine?: number };
  shengKe: { summary: string; tip: string; whyJi?: string; whyYuan?: string };
  themeWord: string; // 锚点用，取自 primary.keywords[0] 或场景压缩词
};
```

来源：现有 `CastResult`、`composeScene`、`shiYingRelation`、`resolveYongShen`、`buildShengKeGraph` / `shengke-map`、`LINE_ROLE`（爻位角色）。

## 速断渲染 `renderQuick*`

### 盘面

- 左右对照表（上六爻在上），列至少：爻象、爻位、六亲、世/应/动标记
- 表头：本卦名 / 变卦名；顶栏可保留四柱摘要（若深度已有装卦时间）
- 不在盘面下堆长文解释

### Tab「结论」

- 1 句总判：本卦 gist 压缩 +（有变则）「→ 变卦关键词」
- 可选 1 行结构标签：`世应{rel}` · `动爻 n` · `本→变`（点名即可，不展开因为所以）

### Tab「分类建议」

短条生成（模板，非古文照搬）：

| 条 | 规则 |
|---|---|
| 运势 | `primary.gist` 一句 |
| 事业 | `composeScene.career` 压成 ≤40 字，或 strategy 事业项 |
| 感情 | `composeScene.love` 同上 |
| 决策 | 有动：朝变卦 keyword 做一小步；无动：先稳本卦边界 |

问题域命中时，对应条置顶或加「本题相关」标记。

### Tab「原文」

- 折叠：卦辞 / 大象 / 动爻爻辞（复用 `classic-corpus` + `classic-gloss`）
- 默认收起；动爻行可默认展开对应爻辞

## 学习渲染 `renderLearn*`

### 文案铁律

1. 句式优先：`因为…，所以…` / `因为动爻在…，具体会发生…`
2. **允许**写「世＝你、应＝对方/环境」作入门锚
3. **禁止**停在定义：锚之后必须接爻位角色 + 本卦生活映射
4. 同一事实在速断/学习语义一致，只是详略不同

### Tab「此刻解读」

```
Step1 锚点：你今天的问题核心在于「{themeWord}」。
Step2 推导：
  - 因为你问的是…，上下卦是「{lowerNature}+{upperNature}」→ 场景翻译
  - 因为动爻在 {labels}（n 个），把「{primary}」推成「{changed}」→ 变化含义
  （无动：因为没有动爻，格局先停在本卦 → …）
Step3 结论：所以策略是…（由本→变 + 世应关系模板）
↓
行动建议：现有 strategy pack 列表挂在结论下
```

删除首页「五步速览只列名词」作为主叙事；五步结构可保留在「怎么读卦」。

### Tab「核心要素」

每个块结构：`定义锚` → `爻位映射` → `对本题含义`

示例骨架：

- **世**：世＝你。世在{label}（{role}）→ 映射句（触顶/卡住/开端…）
- **应**：应＝对方或环境。应在{label}（{role}）→ 映射句
- **用神**：按问题域取六亲；写清是否与动爻咬合；无问题则提示先写下所问

视觉：可保留世应高亮圈，但主文是映射句，不是卡片标题式定义。

### Tab「怎么读卦」

步骤标题改为口语指令，例如：

1. 先找「你」和「外界」的位置  
2. 看看哪些地方「动」了？（找转机）  
3. 把上下两个卦翻译成生活场景  

每步内容：

- **【本卦实战】**：一句场景 + 一句行动指令（模板生成，直接展示）
- **【变卦实战】**（仅有变卦时）：**选择题** → 用户点选 → 再揭正确释义与行动指令  

#### 变卦实战选择题规则

- 出 **3 个选项**（1 正确 + 2 干扰）
- 正确项：由变卦 keywords / gist / 上下卦场景压缩成「正确读法」一句
- 干扰项：从固定干扰池抽取，例如「应立刻硬冲」「完全躺平不管」「只看本卦忽略变化」等，且不得与正确项同义
- 交互：点选后立刻反馈对错；无论对错都展开「正确释义 + 行动指令」
- 不记分、不阻断后续步骤；可「再试一次」重置选项态

### Tab「深度推演」

1. **装卦表**（现有）  
2. **生克星图**（现有）+ 边旁注「帮/克」+ **原理解释句**  
   - `whyYuan`：为什么此爻是原神（生用神）  
   - `whyJi`：为什么此爻是忌神（克用神）+ 结合动爻/问题的提醒  
3. **古文三层**（每段卦辞/大象/相关爻辞）：  
   - 原文  
   - 白话翻译（`classic-gloss`）  
   - 现代场景映射（由 domain + scene + gist 模板生成）  
4. **FAQ**：保留现有问；**新增**至少一问：  
   - 「为什么这个卦告诉我要观察 / 不能硬冲？」（文案随本→变动态选模板）  
   - 回答必须含本→变因果，而不是只复述卦名

## 错误与边界

| 情况 | 行为 |
|---|---|
| 无问题文案 | 锚点用卦 keywords；用神块提示「写下问题后用神更准」 |
| 无动爻 | 推导第二句走「无变」模板；变卦实战整块隐藏 |
| 用神未落到具体爻 | 星图与 why 句提示回核心要素看定义；不编造爻位 |
| 古文缺 seed | 原文 Tab / 三层：gist + 通用白话，不留空白坑 |

## 验收标准

1. 同一卦：速断结论与学习结论语义一致，学习明显更长且含「因为…所以…」  
2. 速断 Tab 仅三枚：结论｜分类建议｜原文；无「怎么读卦／深度推演」  
3. 学习「核心要素」同时出现定义锚 + 爻位生活映射  
4. 有变卦时，「怎么读卦」出现 3 选 1；选后显示正确释义  
5. 深度 FAQ 含本→变推理问；古文有白话 + 现代场景两层  
6. 现有起卦 / 纳甲 / 生克选取规则回归测试通过

3. **P2**：怎么读卦指令化 + 变卦选择题
